## 🎯 WHAT — 核心问题

AI 生成的中文内容被读者和平台识别率越来越高。根本原因不是内容质量差，而是**统计指纹**：均匀的句长分布、可预测的 n-gram 模式、缺乏"爆发性"（burstiness）。

`lynote-ai/humanize-text`（⭐1069）是当前最完整的开源去 AI 味框架，提供 4 种方法论 + 1 条生产管线。

## 🛠️ HOW — 四种方法

### 方法一：多语言翻译链
```
中文原文 → 英文 → 日文 → 芬兰语 → 回中文
```
每次翻译跨越语言结构差异（日语的 SOV 语序、芬兰语的 15 种格），四轮后原始统计指纹被彻底打散。**适合短文案（<1000字）**。

### 方法二：多轮 LLM 改写
- **轮次 1**：结构变异 — 刻意制造句长不均衡（3词短句 + 40词长句交替）
- **轮次 2**：词汇替换 — 去除 AI 高频词，加口语化表达
- **轮次 3**：上下文精修 — 对照原文保语义
- 关键参数：temperature **1.1-1.3**（高于常规生成的创造性）

### 方法三：检测引导反馈闭环
```
改写 → AI检测器判定 → 标出嫌疑句 → 重点重写嫌疑句 → 再检测 → 通过后输出
```
用 Binoculars + RoBERTa 分类器做多信号检测，精确打击而非盲目改写。**最精确但部署成本高**。

### 方法四：混合翻译引擎
同时用 Google Translate + DeepL + Niutrans 三种引擎翻译，在句子级择优拼接。利用不同引擎的"分布偏移"避开单一指纹。

### 标准管线（v1.5，推荐）
```
输入 → DeepSeek(中文改写, temp 1.3) → DeepSeek(日文改写) → Google(→芬兰语) → Niutrans(→回中文) → 输出
```
四步链，固定参数。**50 对文本测试：信息保留 100%，专家评分 9.1/10**。

## 🔄 APPLICATION — NΞXUS 集成方案

### 对知乎好物推荐的直接改进

1. **Prompt 层面**（零成本，立即生效）：
   - temperature 设为 1.2（打破均匀 token 分布）
   - 写入系统提示："句长要参差不齐。偶尔 3 字短句，偶尔 50 字长句。不要用'首先其次最后'"
   - 写好后再让模型以"暴躁网友"风格重写一轮

2. **后处理规则**（方法三的思路，无模型依赖）：
   ```
   AI 高频词替换表（中文）：
   值得注意的是 → 删掉
   综上所述 → 所以
   在某种程度上 → 有点
   不仅…而且 → 不光…还
   极大地 → 特别/贼
   有效地 → 管用
   提供了 → 给了
   具有…的特点 → …就是…
   ```

3. **管线化**（如有 API 预算）：
   - 首先生成 → 中文→英文(DeepL)→日文(Google)→回中文(Niutrans)
   - 适用于批量生产场景

### 检测方法
写作后自查 AI 味：
- 数句长：连续 5 句以上长度相近（±5 字）= 危险
- 看开头：每段以"首先/另外/此外/总之"开头 = 危险
- 查词频："值得注意的是"/"在当今"/"随着…的发展"出现 = 删除
